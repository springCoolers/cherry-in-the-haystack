## 사용자 작업 스타일 주의사항

1. **코드 수정 전 반드시 허락 받기** — "수정해도 될까요?" 물어보고 "그래/수정해" 답변 받은 후에만 수정
2. **읽기는 자유, 쓰기는 허락** — 파일 읽기/조사는 마음대로, 코드 수정은 반드시 승인 후
3. **임시방편 금지** — 근본 원인을 찾아서 제대로 수정. "때우기식"으로 하면 안 됨
4. **가치 판단 금지** — "불필요하다", "안 해도 된다" 등 개발자 판단은 사용자가 함
5. **기획 먼저** — 큰 기능은 기획서 작성 → 검토 → 승인 → 구현 순서
6. **서비스 관점** — 로컬 개발 편의가 아닌, 실제 서비스 사용자 관점으로 설계
