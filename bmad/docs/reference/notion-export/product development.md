# **Overview**

- **핵심 가치:**
    1. 공개 소스를 **소스 탐색 →**
    2. **수집(RSS/크롤링) → 1차 적재(Raw DB) →** 
    3. **정제+AI 초안 → 싱크DB(노션 연동) →** 
    4. **노션에서 배정· 개인리뷰 →** 
    5. **주간 협력 리뷰(보정, review done) →** 
    6. **주간 결과 싱크DB 반영 → Weekly Update(한 주 요약, 주피터북)** 
    7. **→ 월간 리뷰(승격/강등, Patchnote 자동)**의 흐름으로 운영
- **핸드북 메뉴 매핑(최종 플로우):**
    - **다이제스트 → 한 주 요약:** 매주 수요일, 그 주 **review done** 항목들을 **점수로 정렬**해 주피터북에 업데이트
    - **NEWLY DISCOVERED:** “한 주 요약”에 모인 항목 중 **유용 판정(TBD)** 을 받은 것들을 이동(카테고리 미정 시 ND에 우선 적재; **하위 14개 폴더는 추후 제공**)
    - **LLM Engineering – 기초 / ADVANCED:** **월 1회 월간 리뷰(특정 리뷰어 반드시 포함)** 로 ND/기존 항목을 승격 또는 강등하여 반영
    - **Patchnote:** 섹션 이동/추가/삭제 변경은 **자동 기록**(diff 기반)
- **세부 정책(분석 방식/소스/스키마/권한 등)은 현재 모두 미정(TBD)** 이며, 최소 동작을 우선 구현

# **Problem**

- 정보량과 속도가 인간의 수용 범위를 초과
- 2차/3차 재유통 채널만 보면 느리고 불완전
- 놓치면 학습·제품 전략에 손실

# **Solution**

### **1. 백엔드 수집·정제 시스템(단순 흐름, 세부는 TBD)**

- **소스 탐색:** 후보 수집·승인 목록을 만든다(TBD)
- **데이터 수집:** RSS는 **TTRSS**, 비RSS는 **크롤러**로 수집(도구/주기/에러 처리 TBD)
- **1차 적재:** 수집된 원문은 **Raw DB** 테이블에 **가공 없이 저장**
- **정제+AI 초안:** 간단 정제(방식 TBD) 후 **AI Summary / Why it matters** 초안을 생성하여 **싱크DB**에 기록. 싱크DB는 이후 **노션 DB와 연동**된다

### **2. 커뮤니티 협력 리뷰(노션 중심, 규칙은 최소)**

- **담당자 배정 & 개인 리뷰:** 노션에서 담당자를 배정하고 **개인 점수 포함** 리뷰 작성(배정 규칙/점수 기준 TBD)
- **주간 협력 리뷰:** 주 1회 리뷰로 **그룹 점수 보정** 및 내용 확인, **review done**으로 표기
- **싱크백:** 주간 리뷰 결과(보정 점수/상태)는 **싱크DB**에 **업데이트**

### **3. 배포 & 가시성(주피터북/관찰)**

- **Weekly Update(한 주 요약):** 해당 주 **review done** 항목을 **점수 정렬**로 모아 **주피터북**에 업데이트
- **NEWLY DISCOVERED 이동:** “한 주 요약”에서 **유용 판정(TBD)** 된 항목은 **ND**로 이동(카테고리 없으면 ND에 우선 적재)
- **월간 리뷰(특정 리뷰어 포함):** ND/기존 항목을 **기초/ADVANCED** 로 승격 또는 강등 결정(매핑 불가 시 ND 유지)
- **Patchnote 자동화:** 승격/강등/이동 변경은 **Patchnote**에 자동 기록
- **Observability:** **Metrics · Logs · Alerts · Dashboards** 형태로 최소 지표부터 수집(항목/임계치 TBD)

# **성공 기준 (Success Criteria)**

**성공 기준에 대한 정책 필요 : 기준 변경 가능**

1. 매주 **weekly 리포트(다이제스트–한 주 요약)** 발행
2. 조회수/구독자 수 **지속 증가**

5. 자동화 인프라 완성: **수집→정제→리뷰 대기→분류/배포(핸드북/뉴스레터)**

# **노션**

> 아래 필드/명칭/타입은 **임시**이며, 실제 정의는 **TBD**
> 
- **💽 Data Sources:** 소스 카탈로그(종류/인증/빈도/상태)
- **🛡️AI 정보: 찾은 컨텐츠:** 메인 DB(수집·정제·초안·리뷰·출판 플래그)
- **People:** 담당자(역할/전문태그/선호 소스/주간 처리량)

## **핵심 필드(최소안, TBD)**

- **수집:** source_id, url, published_at, ingested_at
- **정제+초안:** normalized_text, ai_summary, why_it_matters
- **연동키:** item_id(싱크DB) ↔ notion_page_id(노션)
- **리뷰:** assignee, score_individual, score_group, status(waiting|review_done), notes
- **출판/월간:** weekly_digest_week_key, nd_candidate(bool), monthly_decision(basic|advanced|nd|none)

## **상태 머신(최종 플로우 반영, 단순형)**

ingested(Raw DB) → refined(Sync DB) → synced_to_notion → waiting(개인) → review_done(주간) → synced_back_to_syncdb(주간 결과 반영) → monthly_decided

## **점수/선정 규칙(최소, TBD)**

- **한 주 요약:** 해당 주 review_done 항목을 **점수 정렬**로 수록(임계값 유무는 추후)
- **ND 이동:** “한 주 요약”에서 **유용 판정**된 항목(정량 기준은 TBD)
- **월간:** ND/기존에서 **기초/ADVANCED** 로 승격/강등

## **System Flow (요약 텍스트)**

1. **소스 탐색** → 승인 소스 목록(간단 목록, 기준 TBD)
2. **데이터 수집**(RSS/크롤링) → **1차 적재: Raw DB**
3. **정제+AI 초안** → **싱크DB(노션 연동)** 적재
4. **노션에서 담당자 배정 & 개인 리뷰(점수 포함)**
5. **주 1회 협력 리뷰:** 그룹 점수 보정, 요약·의미 확인, **review done**
6. **주간 결과 싱크DB 업데이트**(보정 점수/상태 등)
7. **Weekly Update(한 주 요약):** 그 주 review done 항목을 **점수 정렬**해 **주피터북**에 반영
8. **ND 이동 결정:** “한 주 요약”에서 **유용 판정**된 내용은 **Newly Discovered**로 이동(카테고리 없으면 ND 유지, 하위 14개 폴더는 추후)
9. **월간 리뷰(특정 리뷰어 포함):** ND/기존 항목을 **LLM Engineering 기초/ADVANCED** 로 승격 또는 강등
10. **Patchnote 자동 업데이트:** 섹션 이동/추가/삭제 반영
11. **Observability:** **Metrics · Logs · Alerts · Dashboards** 형태로 최소 지표부터 수집·모니터링

## **System Architecture (운영 정책)**

> 모두 **TBD**로 시작하며, 최소 규약만 합의해 진행
> 
- **정책 가변성:** 초기엔 간단 정책으로 시작(정제/초안/선정/출판), 필요 시 버전 관리 도입
- **배정 공정성:** 담당자 배정 규칙은 단순 규칙으로 시작(예: 로테이션), 세부 가중치/부하 균형은 추후
- **권한/RACI:** 노션/깃허브 권한은 최소 롤(에디터/리뷰어/관리자)부터
- **아카이브:** 저점수/오래된 항목 아카이브 기준은 추후
- **월간 리뷰:** **특정 리뷰어 반드시 포함** 조건을 체크리스트/필드로 기록
- **Patchnote:** 파일 이동/추가/삭제만 우선 기록(사소 수정 제외 규칙 TBD)

### **변경/충돌 사항 정리**

- **이전 “≥3점 자동 포함” 규칙은 제거**: 지금은 **그 주 review done 전체를 점수 정렬**로 “한 주 요약”에 수록(임계값 사용 여부는 **TBD**)
- **NEWLY DISCOVERED 이동은 “한 주 요약”에서 유용 판정**으로 결정(5점 고정 규칙 아님)
- **월간 리뷰는 특정 리뷰어 포함**이 필수이며, **기초/ADVANCED** 로 승격/강등 결정
- **카테고리 미정 항목은 ND 유지**, ND 하위 14개 폴더는 **추후 제공**
- **월간 리포트는 작성하지 않음**(삭제)
- **Patchnote 자동화는 유지**, 섹션 이동/추가/삭제를 기록
